# Git Safety Rules

- Niemals destruktive Git-Kommandos wie `git reset --hard`, `git checkout --`, `git push --force` oder ungefragtes `git clean -fd` verwenden.
- Bestehende Nutzer-Aenderungen nicht stillschweigend zuruecksetzen.
- Vor Aussagen ueber den Status geaenderter Dateien immer den aktuellen Zustand pruefen.
- Branch- oder Delivery-Aussagen nur machen, wenn der aktuelle Git-Zustand dazu passt.
