# Clean Code Rules

Diese Regeln gelten fuer alle Codeaenderungen in diesem Repository.

## Grundprinzipien

- bevorzuge kleine, klar abgegrenzte Aenderungen
- schreibe selbsterklaerenden Code mit sprechenden Namen
- halte Funktionen, Komponenten und Methoden fachlich fokussiert
- vermeide unnoetige Kopplung und versteckte Seiteneffekte
- bevorzuge klare Kontrollfluesse statt cleverer Kurzloesungen

## Warnsignale

- ueberladene Funktionen mit mehreren Verantwortlichkeiten
- implizite Datenumformungen ohne klare Benennung
- unmotivierte Sonderfaelle ohne Absicherung
- Kommentare, die schlechten Code nur erklaeren statt ihn zu verbessern
